LLMText Format

Compressed, LLM-optimized output format for efficient token usage and analysis.

Overview

The LLMText format is designed specifically for Large Language Model (LLM) interactions, providing a compressed representation of code structure that minimizes token usage while preserving essential information.

Key Features

  • Compressed syntax - Reduces token count by up to 70% compared to JSON
  • Pipe-separated structure - Easy to parse for both humans and AI
  • Essential information preserved - Node types, names, and positions
  • Perfect for AI context - Optimal format for codebase analysis

Format Structure

The LLMText format uses pipe-separated values with the following structure:

filepath|node_type|child_node_type:name[start_pos]|...

Usage

# Generate LLMText output
code-outline "src/**/*.ts" --format llmtext

# Save to file for LLM processing
code-outline "src/**/*.ts" --format llmtext > codebase.llmtext

# Combine with depth limiting
code-outline "src/**/*.ts" --format llmtext --depth 3

Example Output

Given this TypeScript file:

// example.ts
export class UserService {
  constructor(private db: Database) {}

  async getUser(id: string): Promise<User> {
    return this.db.users.findById(id);
  }

  async createUser(data: UserData): Promise<User> {
    return this.db.users.create(data);
  }
}

export function validateEmail(email: string): boolean {
  return /^[^\s@]+@[^\s@]+\.[^\s@]+$/.test(email);
}

The LLMText output would be:

example.ts|program|export_statement[0:0]|class_declaration:UserService[0:13]|method_definition:constructor[1:2]|method_definition:getUser[3:2]|method_definition:createUser[7:2]|export_statement[12:0]|function_declaration:validateEmail[12:16]

Comparison with Other Formats

Format Token Count Human Readable LLM Optimal
JSON ~500 tokens ✅ Yes ❌ Verbose
YAML ~350 tokens ✅ Yes ⚠️ Moderate
ASCII ~200 tokens ✅ Yes ⚠️ Visual
LLMText ~150 tokens ⚠️ Compact ✅ Optimal

Use Cases

  • AI Code Analysis - Provide codebase context to LLMs efficiently
  • Code Documentation - Generate compact structure summaries
  • Automated Reviews - Feed structure data to AI review systems
  • Code Understanding - Help AI understand large codebases quickly

Integration Examples

# Use with ChatGPT CLI
code-outline "src/**/*.ts" --format llmtext | chatgpt "Analyze this codebase structure"

# Save for later AI processing
code-outline "src/**/*.ts" --format llmtext > codebase-context.txt

# Combine with other tools
code-outline "src/**/*.ts" --format llmtext | your-ai-tool --analyze
